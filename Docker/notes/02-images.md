# Docker Images and Dockerfiles

A hands-on introduction to Docker Images, Dockerfiles, and Containers, including building and running a Java application inside a Docker container.

## 🎯 Docker Image Lifecycle

```text
Dockerfile → Docker Image → Docker Container
```

* **Dockerfile** contains instructions for building an image.
* **Docker Image** is a packaged blueprint of an application and its dependencies.
* **Docker Container** is a running instance of a Docker Image.

---

## 📦 What are Docker Images?

A Docker Image is a lightweight, executable package that contains everything required to run an application:

* Application source code
* Runtime environment
* Libraries and dependencies
* Configuration files

Docker Images are created using a **Dockerfile**, which contains a set of instructions that Docker follows to build the image.

---

## 🏗️ Understanding Dockerfile

A Dockerfile is a text file that defines the steps required to build a Docker Image.

### Common Dockerfile Instructions

| Instruction  | Purpose                                               |
| ------------ | ----------------------------------------------------- |
| `FROM`       | Specifies the base image                              |
| `WORKDIR`    | Sets the working directory inside the container       |
| `COPY`       | Copies files from host machine to container           |
| `RUN`        | Executes commands during image build                  |
| `CMD`        | Defines the default command when the container starts |
| `ENTRYPOINT` | Defines a fixed startup command                       |

### CMD vs ENTRYPOINT

#### CMD

* Provides a default command for the container.
* Can be overridden when running the container.

Example:

```bash
docker run my-image echo "Hello"
```

#### ENTRYPOINT

* Defines the main executable of the container.
* Cannot be easily overridden through normal runtime arguments.

---

## 📝 Dockerfile Example

```dockerfile
# Pull a base image containing Java

FROM openjdk:17-jdk-alpine

# Create a working directory inside the container

WORKDIR /app

# Copy application source code

COPY src/Main.java /app/Main.java

# Compile the Java application

RUN javac Main.java

# Execute the application when the container starts

CMD ["java", "Main"]
```

## ⚙️ Commands Used

### Login to Docker Hub

```bash
docker login
```

Authenticates Docker with your Docker Hub account.

---

### Pull an Existing Image

```bash
docker pull hello-world
```

Downloads the `hello-world` image from Docker Hub.

---

### View Available Images

```bash
docker images
```

Lists all images stored locally.

---

### Run a Container

```bash
docker run hello-world
```

Creates and starts a container from the image.

---

### Stop a Running Container

```bash
docker stop <container_id>
```

Stops a running container.

---

## 🔨 Building a Custom Docker Image

### Create a Dockerfile

```bash
vim Dockerfile
```

### Verify Dockerfile Contents

```bash
cat Dockerfile
```

### Build the Image

```bash
docker build -t java-app .
```

* `-t` assigns a tag (name) to the image.
* `.` tells Docker to use the current directory as the build context.

---

## 🔍 Container Inspection Commands

### View Container Logs

```bash
docker logs <container_id>
```

Displays output generated by the container.

### Attach to a Running Container

```bash
docker attach <container_id>
```

Connects your terminal to the container's main process.

### Open an Interactive Shell

```bash
docker exec -it <container_id> sh
```

Or:

```bash
docker exec -it <container_id> bash
```

Allows executing commands inside a running container.

---

## 📚 Key Learnings

* Docker Images contain everything needed to run an application.
* Docker Images are created from Dockerfiles.
* Containers are running instances of Images.
* `FROM`, `WORKDIR`, `COPY`, `RUN`, and `CMD` are fundamental Dockerfile instructions.
* `CMD` can be overridden during runtime, while `ENTRYPOINT` defines the primary executable.
* Docker Hub can be used to pull pre-built images.
* Custom images can be built and executed using Dockerfiles.
